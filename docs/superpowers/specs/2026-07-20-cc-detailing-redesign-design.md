## Design read

This is a full visual overhaul of a local detailing site for everyday and enthusiast vehicle owners. It uses a premium automotive-editorial language rather than a generic luxury-template language.

- Design variance: 8. Asymmetric poster composition with controlled cropping and generous negative space.
- Motion intensity: 5. Visible but restrained motion for hierarchy and interaction feedback.
- Visual density: 4. Concise sections, large photography, and practical service information.

## Goals

1. Make C&C feel skilled, credible, and locally relevant within the first viewport.
2. Keep the real services, pricing, appointment notes, and contact details easy to scan.
3. Make authentic vehicle photography the primary proof.
4. Give visitors a clear path to request a quote from any page.
5. Deliver a deliberate desktop and mobile experience without template-like visual filler.

## Visual system

### Color

- Canvas: `#070707`
- Raised surface: `#101010`
- Divider: `#292929`
- Primary text: `#F2F0EB`
- Secondary text: `#AAA7A1`
- Muted text: `#77736F`
- Signal red: `#DC2B3E`
- Signal red hover: `#BB2334`

The site stays in one dark theme. One solid red final CTA band is allowed as an intentional interruption. No warm-paper testimonial section, decorative gradients, glows, glass panels, or alternate theme sections.

### Typography

Archivo is the only type family. Use weight, scale, crop, and spacing to create hierarchy.

- Hero: 68-92px desktop, 42-54px mobile
- Page title: 56-76px desktop, 40-50px mobile
- Section heading: 38-58px desktop, 30-40px mobile
- Body: 16-19px
- Navigation and short functional labels may use uppercase. Section headings and marketing copy use sentence case.

### Shape and material

- Sharp corners throughout, with a maximum 4px radius on inputs where needed for usability.
- No floating cards. Group content with spacing, grid, and sparse hairlines.
- Buttons are rectangular and compact. Primary buttons use signal red; secondary actions are text links or thin outlines.
- Use one icon family only. Icons appear only when they improve comprehension.

### Photography

- Use authentic C&C images from the existing site and supplied Facebook page wherever accessible.
- The hero uses a wide, dark vehicle image with a calm area for text.
- Portfolio imagery is never labeled with invented vehicle names, treatments, or outcomes.
- If a factual caption cannot be supported, show the image without a caption.
- Do not use stock imagery in the final portfolio. Temporary implementation placeholders must be clearly isolated and replaced before launch.

## Motion

1. Hero copy enters in a short stagger while the image settles from a subtle 1.03 scale. This establishes hierarchy.
2. Service names and prices reveal as the index enters the viewport. The single interrupting image uses a restrained crop shift. This connects services to the visual result.
3. Portfolio images reveal with a clipped upward motion and use a slight crop shift on hover. This reinforces that the gallery is interactive.

Only `transform` and `opacity` animate. All motion respects `prefers-reduced-motion`. No continuous decorative animation, custom cursor, horizontal scroll hijacking, or generic parallax.

## Technical architecture

- Vite, React, and TypeScript
- React Router for `/`, `/portfolio`, and `/contact`
- Native CSS with shared design tokens and component-level class naming
- Local data modules for services, contacts, navigation, and gallery assets
- Reusable `SiteHeader`, `SiteFooter`, `PageTransition`, `QuoteCTA`, and `GalleryLightbox` components
- Route components stay composition-focused; business facts live in a single typed data source
- Static deployment compatible with Vercel or equivalent hosting

## Data and interaction flow

- The shared business-data module feeds service prices, phone links, email links, notes, and CTA labels across all routes.
- Gallery items are loaded from a typed local manifest. Missing optional captions do not create empty UI.
- Contact form state is local to the Contact page. Validation happens before any mail client handoff.
- Navigation closes after route changes and restores focus appropriately on mobile.
- Lightbox state is URL-independent and resets on route change.

## Error handling

- Broken gallery images are removed from the visible grid and reported in development.
- Contact validation errors appear below the relevant field and are announced to assistive technology.
- If a mail client cannot be opened, the page reveals the email address and a copy action.
- The photo attachment control explains that attachments must be added manually to the email until a production form endpoint exists.

## Accessibility

- WCAG AA color contrast for text, controls, focus states, and form feedback
- Minimum 44px tap targets
- Semantic landmarks and logical heading order
- Skip link and visible keyboard focus
- No placeholder-only form labels
- Keyboard-operable mobile navigation and gallery lightbox
- Useful alt text based only on known image content

## Responsive behavior

- Desktop verification: 1440px and 1024px
- Tablet verification: 768px
- Mobile verification: 390px
- High-variance desktop layouts collapse to a deliberate single column below 768px.
- Hero uses `min-height: 100dvh`, never `100vh`.
- CTA labels remain on one line.
- Navigation remains one line on desktop and never exceeds 72px in height.
